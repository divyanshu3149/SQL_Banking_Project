# 🏦 Banking Data Analysis using SQL

## 📌 Project Overview

This project demonstrates the use of SQL for designing a relational database and performing advanced data analysis on a banking dataset.

## 🗂️ Database Design

The database consists of three main tables:

* **customers** → Stores customer details
* **accounts** → Stores account information
* **transactions** → Stores transaction records

Relationships:

* One customer → Multiple accounts
* One account → Multiple transactions

## 🛠️ SQL Concepts Used

* Basic Queries (SELECT, WHERE, ORDER BY)
* JOINs (INNER JOIN, LEFT JOIN)
* Aggregations (SUM, COUNT, AVG)
* Subqueries
* CASE Statements
* Window Functions (RANK, ROW_NUMBER, Running Total)

## 📊 Key Analysis Performed

* Customer segmentation based on transactions
* Account-level transaction analysis
* Fraud detection (high-value transactions)
* Running balance calculation
* Ranking customers based on transaction amounts

## 📈 Sample Insights

* Identified high-value customers based on transaction volume
* Detected unusual withdrawal patterns
* Analyzed monthly transaction trends

## 🚀 Conclusion

This project highlights the ability to design databases and perform real-world financial data analysis using SQL.
